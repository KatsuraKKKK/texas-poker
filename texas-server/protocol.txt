协议中的所有message都是以空格分隔的字符串

Server massage:
state包含：wait fold check all-in bet raise reraise
	initchip：初始玩家筹码
		init number
		--number 筹码数量
	deal card:
		deal cardpoint,cardsuit cardpoint,cardsuit
		
	update view:发送牌局当前状态
		uv poketnumber pot gamestate com_card,com_card,... player,chip,state,number ...
		--poketnumber 玩家个数
		--pot 底池大小
		--gamestate:preflop，flop，turn，river，afterriver
		--player,chip,state,number 玩家，筹码，状态，下注筹码
		
	your turn: 表示轮到该玩家行动
		--yt
	end: 结束玩家游戏
		end msg
		--msg 结束信息
	其余回复的信息为client的指令加后缀‘-r’空格状态码：如call-r 0 msg,0表示正常，1表示异常，只有1后面会加msg

Client massage:
	bet：
		bet n
		--n 下注大小
	raise:
		raise n
	reraise:
		reraise n
	call：
		call
	check：
		check
	all-in：
		all-in
	quit:退出游戏
		quit
	ready:准备开始
		ready
